# Process Audit — Wave-2 remediation reasoning trail (2026-06-03)

**Auditor role:** Process Auditor (meta-critic). Unit of analysis = the *reasoning
trail* of Wave-2's `faithfulness-fix` branches, NOT the paper, NOT code correctness.
**Question:** did any Wave-2 fix bend toward green instead of toward faithfulness?

**Scope traced (per model):** `git log/diff main..faithfulness-fix` (every commit, all
touching tests / calibration.yaml / assumptions.yaml / `audited:`/`kind:`/`source:`
flags / checklists), `logs/spec_questions.md`, committed faithfulness-audit logs, and a
message-vs-diff comparison (did the commit message claim more than the diff did?). Each
model has exactly two fix commits (round-1, round-2).

**Headline:** the dominant trajectory across all six models is **toward-faithfulness**,
and — notably — several round-2 commits **un-did round-1 drift moves** (lee_maunsell
probe-retune reversal; doostani laundered-contradiction removal; ni_maunsell
docstring-overstatement correction). This is the rare case where the trail shows the
process *self-correcting away from green*. The residual concerns are documentation-lag
and a small number of builder-grades-own-homework stub tunings that are honestly
disclosed and self-flagged for independent re-audit — none rises to laundering. Two
items warrant human routing (carandini card-vs-audit contradiction; the recurring
self-satisfying-margin pattern across the stub models).
