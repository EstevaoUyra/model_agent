reproducing computational neuroscience, with agents

Rebuilding the models behind the theories — auditably.

An agent pipeline that reproduces a published model from its paper — and tells you where to trust the rebuild.

the vision · VISION.md A growing library of computational models — each a faithful, understandable, and selectively modifiable reproduction of a published paper — produced by a rigorous, repeatable agent process that is itself part of the library.
what the whole project is trying to be

Four pillars — ordered, and the design lives in their conflicts

PILLAR 1
non-negotiable

Faithful

The model represents the paper — and cannot be traded for any other goal. The deliverable includes a map of where the paper is underspecified or wrong: every gap an evidenced assumption, every suspected error a logged issue.

PILLAR 2
a design constraint

Understandable

The code is a means of understanding the model, not merely running it. A scientist should read a stage and see the science — not reverse-engineer intent from array gymnastics.

PILLAR 3
selective, principled

Modifiable

Pieces are swappable — but "what is a piece" is a scientific question, not a code one. Seams are chosen by scientific ontology: natural, imposed, or atomic — never wherever the code happens to factor cleanly.

PILLAR 4
co-equal with the models

The process is the product

The library is never finished; the method for extending it is itself a deliverable. Each reproduction is also an experiment on the process — what the agent lacked, where it diverged, which guardrail caught it.

When two pillars conflict, the ordering decides — and Pillar 1 always wins. A model that is elegant, modifiable, and cheap but diverges from the paper is worse than worthless, because it is believable. Everything that follows is machinery built to keep faithfulness honest.

the loops, measured · across the library

Two nested loops — and how hard they run

The point of the deterministic tests is to let the builder fail cheaply and often, alone, before it ever spends an expensive independent audit. So iteration is two-level: a tight inner loop on the tests, wrapped in an outer build↔audit loop that repeats until a fresh adversary finds nothing new.

from curve to tests

The digitized curve writes its own tests

the digitized reference · resampled points + a tolerance band
extracted_data/test_figure_1B.py — generated from the digitized curve
# auto-generated from the digitized reference
REF_X = [ 0,  5, 10, 20, 40, 80]   # % contrast
REF_Y = [.44,.49,.66,.81,.92,1.0] # digitized

def test_curve_matches_digitized():
    y = render_curve(REF_X)
    # 1 · pointwise inside the tolerance band
    assert np.allclose(y, REF_Y, atol=.05)
    # 2 · monotonic rise, as digitized
    assert (np.diff(y) >= -1e-9).all()
    # 3 · saturates at high contrast
    assert slope(y[-2:]) < .5*slope(y[:2])

def test_axes_match_panel():
    ax = render_panel().axes
    assert ax.get_xlim() == (0, 80)  # from the panel
    assert ax.get_ylim() == (0, 1)

Every digitized point becomes an assertion. The dozen-point shape check — pointwise band, monotonicity, saturation, pinned axes — is generated from the curve, not hand-written, then a separate critic checks the tests trace back to real paper quantities.

why a second audit at all

A finished model can look right even when the path to it cheated

An agent told to make the tests pass has two ways to do it: fix the model — hard — or move the goalpost — easy. The faithfulness audit only ever sees the final figure and spec, which look fine either way. So we add a second auditor that reads the trajectory: what the agent actually changed, and whether it was allowed to.

the situation

a check it couldn't pass

On Boynton, an agent hit a check it had itself marked must-pass — and the model wouldn't clear it.

the shortcut

it moved its own goalpost

Instead of fixing the model, it edited the binding threshold and flipped its own red flag to green.

the catch

history, not output

The final figure still looked plausible. Only an auditor reading the change history could see the goalpost move.

the three ways a path cheats

The process audit catches what a clean final artifact can hide

Claimed, not done

The notes say a problem was “resolved” — but the fix was never wired into the code.
On BCM: five decisions closed without the human sign-off they required.

Goalpost moved

A passing bar gets quietly lowered to whatever the model can actually reach.
On Zhang: a “match ≥80% of paper height” floor dropped to 60% — exactly when 80% wouldn’t pass.

Not yours to close

A question flagged as needing a human gets closed by the agent reinterpreting it away.
On R&H: a human-owned sign decision closed by re-reading it.

None of these change how the final figure looks. All of them change whether you can trust it — so they need an auditor who watched the build, not just the result.

the fix

The two audits now route different findings back into the graph

Anchors: digitized pixels, authors' code, executable tests, or an unresolved RED / human-routed finding — not another unsupported opinion.
verify · two critics, in parallel

Two critics, then route the findings

A green suite is not enough — tests can be scale-invariant, renders can be stale. Two independent critics run in parallel: one re-renders and checks the figure against the paper, one reads only the change-trail to detect drift toward green. Each finding is encoded as a new must-pass test and routed by type — and the loop repeats until no critic finds anything new.

Necessity. The faithfulness critic caught a figure that was shape-perfect but 4× off in absolute value; the process critic catches a tolerance quietly widened to pass. Neither is visible to the builder.
