Each model's `README.md` is GENERATED by `tools/build_model_readme.py models/<name>` — one
fault-tolerant renderer per section; it renders artifacts and never invents prose. Do NOT hand-edit a
model README. The 7 sections + sources:

1. Current exit — `logs/exit.json` (overall/trajectory/flagged_count/blocked/audit/headline; also read by the parent `tools/build_readme.py`)
2. Status — exit.json + live test/VLM roll-up (`logs/test_runs.jsonl` + `logs/figure_comparisons/`) + `logs/readme_meta.yaml:status_narrative` + ⚖️ adjudications
3. Model — `article_aware/spec/model_spec.yaml` + `citations.yaml` + `readme_meta.yaml:model_summary`/`model_equation_refs`
4. Reproduced figures — committed figure views + `readme_meta.yaml:figures` (headline/badge/note/checks); discovers BOTH the `figures_reproduced/figure_N.png` convention AND the `figure_outputs/` + `figure_N/figure_N_source.png` convention, gated to committed files
5. Potential sources — `logs/issues.yaml`
6. Changelog — `logs/changelog.md` `## DATE — summary` headers, else a git-log fallback (excludes the autogen commit so --check is stable)
7. Reproduction cost — `tools/repro_cost.py --markdown`

The irreducible prose lives in three authored artifacts: `logs/exit.json`, `logs/readme_meta.yaml`,
`logs/issues.yaml` (+ optional `logs/adjudications.yaml`). The full-pass finalize step's update-state
agent authors them, then runs the generator. Design + schemas:
`proposals/per-model-readme-autogen-2026-06-15.md`. The pytest plugin captures `@pytest.mark.tier`/`check`
into test_runs.jsonl so §4 check tables self-populate on the next suite run. See [[organizer-doesnt-implement-trust-the-process]].
